# DECISIONS LOG

## Accepted

### D-001
- статус: accepted
- решение: основной стек `Laravel 11 + Blade + Alpine.js + Tailwind CSS`
- причина: подходит под обычный PHP-хостинг и дает быстрый MVP без SPA-слоя

### D-002
- статус: accepted
- решение: роли хранить через `roles` и `user_roles`
- причина: нужна гибкость и поддержка мульти-роли

### D-003
- статус: accepted
- решение: критерии оценивания хранить через `rubrics`, `rubric_criteria`, `rubric_levels`
- причина: это масштабируется лучше, чем фиксированные `K1/K2/...` колонки

### D-004
- статус: accepted
- решение: import contract = JSON `v1.0`
- причина: это самый прямой мост между разбором PDF и доменной моделью Laravel

### D-005
- статус: accepted
- решение: импорт всегда создает assessment в `draft`
- причина: нужен безопасный контроль перед использованием в учебном процессе

### D-006
- статус: accepted
- решение: assessment versions обязательны
- причина: старые результаты должны оставаться воспроизводимыми

### D-007
- статус: accepted
- решение: локальная разработка проверяется на SQLite, а PHPUnit использует in-memory SQLite
- причина: быстрый и стабильный локальный цикл, без порчи рабочей dev DB тестами

### D-008
- статус: accepted
- решение: первый assignment flow делать напрямую `teacher -> student`, без group UI
- причина: это закрывает MVP быстрее и не блокирует дальнейшее расширение на teacher groups

### D-009
- статус: accepted
- решение: GitHub workflow ведется через SSH remote `git@github.com:FreeJam/VPR.git`
- причина: это уже настроено и подтверждено в текущем workspace
