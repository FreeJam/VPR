# PERMISSIONS MATRIX

## Роли
- `admin`
- `teacher`
- `student`
- `parent`

## Ключевые права

### Admin
- управляет пользователями
- назначает роли
- управляет классами и предметами
- создает и редактирует тесты
- видит все импорты
- публикует и архивирует тесты
- управляет достижениями
- видит системный журнал

### Teacher
- редактирует свой профиль
- получает код подключения
- принимает и отклоняет заявки учеников
- управляет своими группами
- создает свои тесты
- запускает импорт
- назначает тесты своим ученикам
- видит результаты своих учеников
- проводит ручную проверку своих назначений

### Student
- редактирует базовый профиль
- подключается к учителю по коду
- видит свои назначения
- проходит попытки
- смотрит свои результаты
- видит свои достижения и прогресс

### Parent
- видит только привязанных детей
- смотрит прогресс и результаты ребенка
- читает комментарии учителя

## Ограничения доступа
- teacher не должен видеть чужих учеников без явной связи
- student не должен видеть чужие попытки и оценки
- parent не должен редактировать учебные сущности
- import run доступен только `admin` и `teacher`
- publish assessment в первой версии лучше оставить только `admin`

## Policy-first подход
Для критичных действий нужны Laravel policies:
- `AssessmentPolicy`
- `AssignmentPolicy`
- `AttemptPolicy`
- `ImportBatchPolicy`
- `TeacherStudentLinkPolicy`
- `ManualReviewPolicy`
